PVE 9.0 快速上手：一台物理机搞定虚拟化系统！上班时用做生产工具，闲时做NAS系统。

发表于： 2025-11-23 分类于： NAS
字数： 2465 阅读：≈ 5分钟 浏览：

一台机器就能完成企业级虚拟化环境搭建——这篇文章，带你零基础玩转 Proxmox VE 9.0！

一、为什么推荐从“一台物理机”开始？

很多朋友一听到“虚拟化”“集群”“PVE”，脑子里马上浮现出一堆服务器、电缆和专业名词。 其实——你完全可以从一台物理机开始！

PVE（Proxmox Virtual Environment）9.0 是基于 Debian 13 + Linux 6.14 内核的最新版本，它不但功能强大（KVM+LXC 双虚拟化、快照、备份、直通显卡），而且安装简单、上手快、界面友好。

本篇教程就是为“小白用户”准备的：

从下载镜像、刻录U盘，到安装系统、创建虚拟机、直通显卡，全流程实操。

你只需一台服务器、一根网线、一只U盘，就能把虚拟化环境跑起来！

二、准备工作

实际部署机器配置

部署机器按自己实际情况来，下面是我们本次部署用到的机器配置：

  • CPU： Intel Xeon 8462Y
  • 内存： 1 TB
  • 显卡： NVIDIA GeForce RTX 4070 × 8
  • 接口说明（从左到右）：键盘、U盘、IPMI 管理口、光口、VGA 显示连接线

系统镜像下载

\1. 打开 PVE 官方网站：

https://www.proxmox.com/en/downloads ；

\2. 选择：“Proxmox Virtual Environment 9.0 ISO Installer”；

\3. 下载完成后，准备制作启动盘。

三、U盘启动盘制作

推荐工具：Ventoy（Ventoy 的好处是：刻录一次后可反复使用，换镜像无需重做。）

下载地址：

https://www.ventoy.net/cn/download.html

具体步骤：

\1. 插入U盘，打开 Ventoy，载入 proxmox-ve_9.0-1.iso，选择「继续试用」，在启动选项下，选择「写入硬盘映像」；

\2. 写入方式选择RAW，点击「写入」 ；

\3. 等待刻录完成（大约 3–5 分钟）

四、安装 PVE 9.0 系统

从U盘启动进入系统安装

\1. 将制作好的启动U盘插入目标服务器，开机后立即按下快捷键（通常是Del，F2, F10, F12等，因主板而异）进入BIOS/UEFI设置界面；

\2. 在启动选项（Boot）中将 U盘设置为第一启动项，保存退出；

\3. 重启后，服务器会从U盘启动，也可以直接找到U盘进入系统安装。

进入图形化界面安装PVE

\1. 重启后，你会看到Ventoy的启动菜单，使用键盘上下键选择 proxmox-ve-9.0-1.iso 然后按回车

\2. 接下来会进入PVE的安装界面。选择Install Proxmox VE (Graphical)并按回车

\3. 阅读并同意许可协议 → 点击「I Agree」

\4. 选择安装磁盘（⚠️ 注意：此磁盘所有数据将被清空）

\5. 设置时区：举例，Country: China；Time Zone: Asia/Shanghai ；Keyboard: US English

\6. 设置 root 管理员密码和邮箱：Password：为root用户设置一个强密码；Confirm：再次输入密码；E-Mail：在原来的基础上随意加个数字即可。

\7. 配置网络：这是非常重要的一步！

  • Hostname: 给你的PVE主机起个名字，例如host1.com.cn
  • IP Address (CIDR)： 输入你准备好的固定IP地址和子网掩码，格式为 IP/掩码位数，例如 192.168.1.100/24（/24 对应子网掩码 255.255.255.0）
  • Gateway： 输入你的网关地址，例如 192.168.1.1
  • DNS Server： 输入DNS服务器地址，例如 8.8.8.8

⚠️ 注意：请确保这个IP地址在你的局域网内是唯一的，且没有被其他设备占用。

\8. 点击 Install 开始安装（约 5–10 分钟）

安装完毕后系统会提示重启，点击reboot。

⚠️ 注意：记得拔掉U盘再重启，否则会再次进入安装界面。

五、登录 Web 管理界面进行配置

安装完成后，在浏览器输入：https://<你的PVE主机IP>:8006

如：https://192.168.1.49:8006

  • 默认用户名：root
  • 密码：你在安装时设置的 root 密码

登录成功后，你就能看到 Proxmox VE 9.0 的管理控制台。

它同时支持 中文版与英文版界面，可在右上角切换语言。

接下来，进行创建集群，并挂载共享存储。

创建集群：

在数据中心中找到集群，点击「创建集群」，填写集群名称如cluster1，然后「创建」即可。

集群挂载共享存储：

在数据中心中找到存储，点击「添加存储」，选择“SMB/CIFS”，填写存储的服务器地址、用户名、密码，“内容”全部选择，确认无误后点击「添加」。

六、创建你的第一个虚拟机

创建虚拟机

\1. 在左侧主机（如 host1）上右键 → 创建虚拟机

\2. 命名虚拟机（如 win10-office）

\3. 选择 ISO 镜像（先将Windows镜像上传到 local 或共享存储）

\4. 配置参数：

  • CPU 核心数：2–4
  • 内存：4–8 GB
  • 硬盘：64 GB 以上（建议 SSD ）
  • 网卡：桥接到你的物理网络（vmbr0）

\5. 最后，确认虚拟机参数，点击「创建」即可。

虚拟机开机

创建虚拟机完成后，右键「开机」。

安装 Windows 系统

\1. 打开虚拟机控制台，会出现提示 “Press any key to boot from cdrom” ，此时一定要按任意键，触发启动iso，否则会提示启动失败

\2. 进入 Windows 安装界面，选择版本

\3. 下一页，选择「自定义安装」

\4. 加载驱动（选择你上传的 virtio 驱动）

\5. 系统识别到驱动器后开始安装（约 5–10 分钟）

\6. 安装完成后即可进入熟悉的 Windows 桌面

七、虚拟机进阶操作

显卡直通

\1. 关闭虚拟机

\2. 在“硬件”页添加 PCI 设备 → 选择显卡 (RTX 4070)

\3. 勾选所有选项：All Functions、ROM-Bar、PCI-Express

\4. 保存后重启虚拟机

直通后，虚拟机可直接调用物理显卡，性能媲美裸机。

USB 设备直通

同样在“硬件”中添加 USB 设备：

\1. 选择“使用 USB 端口”

\2. 选中目标设备（如 U盘）

\3. 点击“添加”即可。

八、虚拟机访问方式（多种远程操作）

日常运维时，管理员可通过 PVE 控制台 直接访问虚拟机，实现快速登录，适合在部署或调试阶段使用。

同时，PVE 也兼容多种远程访问协议，包括 VNC、RDP、SPICE 及云桌面客户端，可满足不同使用场景下的访问需求。

这些远程访问方式的配置方法与使用技巧，我们将在后续《PVE 9.0 实战进阶篇》中详细介绍，敬请期待！

九、实测体验

  • 虚拟机启动时间：几秒钟即可启动成功
  • 打开浏览器 / Office / TXT 应用流畅无卡顿
  • 显卡直通后，可直接运行图形渲染与轻度 AI 推理任务
  • 系统稳定运行 72 小时无报错

对于个人学习、测试、轻办公以及企业运行专业软件，这样一台单节点 PVE 主机都完全够用！

总结

通过本文步骤，你已经从零完成了：

✅ 镜像下载与U盘刻录

✅ PVE 9.0 系统安装

✅ Web登录与界面配置

✅ 虚拟机创建与系统安装

✅ 显卡/USB 直通

✅ 远程桌面访问

一句话总结：一台服务器，一个U盘，你也能搭建出媲美 VMware 的开源虚拟化环境！
